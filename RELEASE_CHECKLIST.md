# GitHub 发布检查清单

## 仓库设置

- [ ] 确认 GitHub 账号或组织及仓库名称
- [ ] 决定公开仓库或私有仓库
- [ ] 选择并添加 `LICENSE`
- [ ] 配置安全问题的私人报告渠道
- [ ] 设置默认分支为 `main`

## 内容与安全

- [ ] 确认没有患者数据、受保护健康信息、凭据或未公开材料
- [ ] 确认没有文献全文或其他无权公开的版权内容
- [ ] 检查 README 中的安装、调用和免责声明
- [ ] 检查中文内部审核说明与当前功能一致
- [ ] 检查中文和英文客户交付规则是否同步

## 版本与验证

- [ ] `VERSION`、`manifest.yaml` 和 README 版本一致
- [ ] 运行 `python tests/validate_repository.py`
- [ ] GitHub Actions 验证通过
- [ ] 创建版本标签，例如 `v2.5.1`
- [ ] 发布说明列出新增功能、兼容性和已知限制

## 上传

- [ ] 初始化 Git 仓库并完成首次提交
- [ ] 添加正确的远程仓库地址
- [ ] 推送 `main` 分支
- [ ] 打开 GitHub 页面确认 README、文件树和 Actions 正常显示
