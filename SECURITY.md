# 安全与敏感数据政策

## 不要公开提交的数据

- 患者级数据、受保护健康信息或可重新识别的信息
- 医院内部数据、未公开临床记录或受限队列
- 数据库、服务器、API、图书馆或机构账号凭据
- 未公开论文、基金标书、审稿材料或合作方文件
- 未获得公开授权的文献全文

请使用合成、匿名化且不可回溯的数据复现软件问题。

## 安全问题报告

公开仓库启用后，应配置 GitHub Private Vulnerability Reporting 或指定私人联系邮箱。涉及密钥、患者数据泄露或权限绕过的问题不要创建公开 Issue。

## 使用边界

LXQ 不是医疗器械或临床决策系统，不应直接用于诊断、治疗或患者管理。任何临床、伦理、监管、统计、病理或遗传结论均需要相应资质人员复核。

## 文献与版权

不得使用 LXQ 绕过付费墙、验证码、身份认证、许可或访问控制。公开仓库中不要提交受版权保护的文献全文。
