# 贡献指南

感谢改进 LXQ。提交修改前请遵守以下规则。

## 修改原则

- 保持证据边界，不添加无法核验的医学、临床或生物信息学结论。
- 不提交真实患者数据、受保护健康信息、访问凭据或未公开研究材料。
- 修改中文或英文客户交付规则时，检查另一语言版本是否需要同步。
- 新增功能应放入对应的 `references/` 文件；只有所有任务都需要的规则才进入核心层。
- 可确定性检查应优先实现为 `scripts/` 中的脚本，并提供验证用例。
- 不要把 README、安装指南或发布说明放进 `lxq/` Skill 目录；仓库级说明放在根目录。

## 提交流程

1. 从新分支开始修改。
2. 更新受影响的版本号、路由和中文内部审核说明。
3. 运行：

   ```bash
   python tests/validate_repository.py
   ```

4. 检查没有生成缓存、测试输出、患者数据或密钥。
5. 在 Pull Request 中说明修改范围、证据依据、兼容性和验证结果。

## 问题分级

- `Blocker`：身份、设计、分析或中心解释失效。
- `Major`：可能实质改变结果、主张或提交决定。
- `Minor`：改善报告、复现或展示，不改变中心结论。

修复文字不能替代底层科学问题的修复。
