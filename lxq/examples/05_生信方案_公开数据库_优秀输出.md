# 公开数据库生信方案：优秀示例

## 任务输入

利用公开转录组数据开发疾病复发风险模型，并设计验证方案。

## 调用强度

`standard`

## 任务模式

`execute`、`explain`

## 功能轨道

`analysis`、`literature`、`delivery`

## 推荐输出结构

问题与结局 → 数据来源 → 纳排与预处理 → 特征/算法 → 防泄漏 → 内外部验证 → 指标 → 可视化。

## 最终示范输出

首先锁定公开队列的纳入标准、复发定义、随访时间和平台信息，不把不同终点直接合并。训练队列完成探针/基因映射、低表达过滤、标准化和批次诊断；外部队列保持独立，不参与特征筛选和超参数选择。

模型训练采用嵌套交叉验证，所有特征筛选、缺失处理和标准化步骤均在训练折内完成。报告C-index或时间依赖AUC、校准曲线、Brier分数和决策曲线；同时提供不同平台、随访定义和临床亚组的敏感性分析。图形包括数据流程图、样本质控、特征稳定性、外部校准和风险分层生存图。

## 质量检查点

- 明确数据和终点兼容性。
- 明确参数、拆分、防泄漏和真实外部验证。
- 不把内部交叉验证称为外部验证。
