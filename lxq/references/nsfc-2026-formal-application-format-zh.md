# 2026 NSFC 正式标书交付格式（中文）

## 定位与优先级

本规则将用户提供的《【模版】2026年度NSFC正式申报书模版.doc》固化为 LXQ 的正式基金标书交付格式。可复用 DOCX 资产位于：

`assets/templates/nsfc-2026-formal-application-template.docx`

该文件是机构工作模板，不等同于国家自然科学基金委在线申报系统的永久官方模板。正式交付前必须核验当年度官方指南、在线字段、申请代码、科学属性、限字、简历和附件要求。出现冲突时，以当年度官方要求为准，并记录 `[POLICY_CHECK_NEEDED: ...]`。

源 `.doc` 的 SHA-256 为 `B889A6ACA0377F8DA6A474A2811F57541F8AD4A7CC7F3597E5DF0E184CDB77E0`；技能内转换后 DOCX 的 SHA-256 为 `4E79DE6B69CAB9D98B9B9471AA880F9EBD859050F6AB05AE3AE32F2F90BAC1FF`。

## 触发条件

用户要求以下任一交付时读取本规则：

- NSFC、国自然或国家自然科学基金正式申报书；
- 基金标书 Word 正式版、完整标书、可打印版或院内预审版；
- “按 2026 NSFC 模板”“按上传模板”“产出这种格式”；
- 将已经写好的基金内容装入正式申报书。

课题筛选、售前方案和内部研究设计可继续使用 14 段客户方案；一旦任务进入“正式 NSFC 标书交付”，必须将内容映射到本文件的五页框架，不得把 14 段 Markdown 直接当作最终标书。

## 强制交付物

1. 以模板资产副本为基础生成的 `.docx` 正式标书；禁止修改技能内母版。
2. 保持封面、评审表、基本情况表、报告正文和申请人简历的顺序。
3. 保持模板要求不得删除或改动的提纲标题及括号说明。
4. 将研究设计、样本量、终点、统计、验证、风险和预算信息映射到正式正文，同时保留内部审计表。
5. 正式交付前运行 `scripts/validate_nsfc_template.py <docx>`，并完成逐页渲染检查。

## 五页结构

### 第1页：封面

- 申报代码、科学属性；
- `2026年度国家自然科学基金`、`正式申报书`；
- 申报类别、申请人、科室、手机、邮箱；
- 申请单位、申报年月；
- 科主任签字。

模板中的医院名称和 `2025年11月` 仅为源文件示例，必须根据实际申请人、机构和申报周期替换；无法确认时使用 `[AUTHOR_INPUT_NEEDED: ...]`，不得擅自保留为事实。

### 第2页：专家评审意见表

保留四个评价维度及 A/B/C 选择：

1. 与项目相关的论文发表情况（研究能力、条件等）；
2. 科学意义及价值；
3. 创新性；
4. 逻辑性、可行性；
5. 综合评价。

交付申请人版本时保持未勾选；只有真实评审人完成评审后才能填写。

### 第3页：基本情况表

保留并填写：

- 项目名称；
- 关键词（3-5个）；
- 研究方向；
- 主要研究领域；
- 申请人姓名、性别、出生年月、职称、学位、科室、E-Mail、手机；
- 项目摘要（限400字）。

摘要需覆盖问题、目标、方法、主要终点或关键评价指标、创新点和预期科学价值，不得填入未经核验的前期结果。

### 第4页起：报告正文

必须原样保留以下提纲并按顺序填充：

1. `（一）立项依据与研究内容（建议8000字以下）：`
2. `1．项目的立项依据（研究意义、国内外研究现状及发展动态分析，需结合科学研究发展趋势来论述科学意义；或结合国民经济和社会发展中迫切需要解决的关键科技问题来论述其应用前景。附主要参考文献目录）；`
3. `2．项目的研究内容、研究目标，以及拟解决的关键科学问题（此部分为重点阐述内容）；`
4. `3．拟采取的研究方案及可行性分析（包括研究方法、技术路线、实验手段、关键技术等说明）；`
5. `4．本项目的特色与创新之处；`
6. `5．年度研究计划及预期研究结果（包括拟组织的重要学术交流活动、国际合作与交流计划等）。`
7. `（二）研究基础（与本项目相关的研究工作积累和已取得的研究工作成绩）；`

内部 14 段研究方案按以下关系映射：背景与临床问题、研究假说映射到第1项；目标、研究内容、关键问题映射到第2项；研究对象、样本量、分组、检测、统计、技术路线、风险与替代映射到第3项；创新点映射到第4项；年度计划、里程碑和预期成果映射到第5项；已核验的前期工作、平台和成果映射到研究基础。

### 末页：申请人简历

保留并按模板格式填写：

- 申请人简历；
- 近五年主持的国家级科研项目；
- 近五年代表性论著（限5篇）；
- 其他除论著外代表性成果（限5项）；
- 专著、授权发明专利和获得学术奖励。

项目、论文、专利和奖励只能来自申请人核验材料。不得自动补齐、推断排名或虚构成果。

## 版式合同

- 页面：A4 纵向，`21.0 cm × 29.7 cm`。
- 页边距：上 `2.54 cm`、下 `1.619 cm`、左 `2.076 cm`、右 `1.886 cm`。
- 页眉距 `1.501 cm`，页脚距 `1.75 cm`；正文页使用居中页码。
- 封面主标题：楷体_GB2312、22 pt、加粗、居中。
- 封面单位、日期和签字：仿宋_GB2312、15 pt、加粗。
- 页面主标题：楷体、16 pt、加粗、居中。
- 正文提纲：楷体、14 pt；一级提纲加粗；模板蓝色提示文字使用 `#0070C0`，内容填充后可按用户或机构要求转为黑色，但不得破坏层级。
- 简历主要标题：楷体、14 pt、加粗；格式说明及成果子标题：楷体、12 pt。
- 表格：保持原合并关系、边框、行高、列宽和垂直对齐，不使用重新绘制的近似表格代替。
- 不使用彩色装饰、宣传式封面、额外页眉或与模板无关的品牌元素。

## 填充与修订规则

1. 始终复制模板资产后再填写，保留母版只读。
2. 优先替换空白占位和示例字段，不重建页面结构。
3. 内容增加导致分页变化时，允许正文和简历自然延展，但封面、评审表和基本情况表应保持独立完整页面。
4. 不得为了维持页数而缩小至不可读字号、压缩行距或截断内容。
5. 不得把计划开展的研究写成已完成的前期基础。
6. 不得自动勾选专家评审等级或代替科主任签字。
7. 正文、摘要、研究设计表、预算和参考文献中的项目名称、样本量、终点、时间和单位必须一致。

## 交付前验证

执行：

```powershell
python scripts/validate_nsfc_template.py <正式标书.docx>
```

然后使用文档渲染流程逐页检查：

- 是否存在文字截断、重叠、乱码或字体替换；
- 表格是否跨页破裂或合并关系改变；
- 封面字段、机构、日期和签字状态是否正确；
- 提纲标题及括号说明是否保留；
- 页码、分页和正文层级是否正确；
- 是否仍有未授权的示例事实或 LXQ 占位符。

结构验证通过不等于科学内容通过；仍需运行基金交付验证、证据核验和正式政策检查。
