# Bioinformatics SOP Output Contract

Use `standard` for a planned workflow and `strict` for audit or reproducibility review.

Specify scientific question, data source/state, sample roster, reference/build, preprocessing, QC, software versions, parameters, seeds, contrasts or model, leakage controls, validation, effect/performance metrics, planned figures, outputs, logs, and rerun criteria.

Do not silently substitute methods, mock data, or unavailable tools. Distinguish a planned SOP from an executed analysis.
