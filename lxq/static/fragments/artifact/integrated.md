# Integrated project or submission

- Build one shared fact base and terminology ledger across literature, grant aims, customer-facing schemes, study-design matrices, data, code, analyses, manuscript, figures, supplements, reviewer responses, and repositories.
- Trace every central claim through `source data -> analysis -> result artifact -> display artifact -> prose -> availability location`.
- Reconcile sample counts, exclusions, endpoint definitions, units, thresholds, directions, identifiers, and version labels across every artifact.
- Treat late-stage edits as propagation risks and rerun affected consistency checks.
- Assign both scientific decision and package readiness; never call a submission ready because only the manuscript prose was reviewed.
